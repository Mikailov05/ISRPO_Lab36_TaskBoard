## 2. Описание проекта  

**TaskBoard** — это интерактивная доска задач с полным CRUD-циклом.  
Приложение позволяет пользователю:

-  Просматривать список задач  
-  Добавлять новые задачи (название + описание)  
-  Менять статус (выполнено / не выполнено) — задача визуально перечёркивается  
-  Редактировать название и описание задачи прямо в интерфейсе  
-  Удалять ненужные задачи с подтверждением  
-  Фильтровать задачи: «Все», «В работе», «Выполненные»

## 3. Инструкция по запуску  

###  Бэкенд (ASP.NET Core Web API)

1. Откройте терминал и перейдите в папку проекта:
   ```bash
   cd Lab36_TaskBoard/TaskBoardApi
   ```

2. Запустите сервер:
   ```bash
   dotnet run
   ```

3. API будет доступно по адресу:  
   `http://localhost:5295`  
   Swagger: `http://localhost:5925/swagger`

###  Фронтенд (HTML/CSS/JS)

1. Откройте файл `frontend/index.html` через **Live Server** (в VS Code) или просто в браузере  
2. Убедитесь, что бэкенд запущен, иначе запросы будут падать с ошибкой

>  Если ваш бэкенд запускается на другом порту — отредактируйте `frontend/js/api.js` (константа `API_URL`).

## 4. Главные выводы по лабораторной работе

1. **Разделение ответственности**  
   Серверная часть (ASP.NET Core + EF Core) занимается только данными и бизнес-логикой. Фронтенд (HTML/CSS/JS) отвечает за интерфейс и взаимодействие с пользователем. Это основа любой современной fullstack-архитектуры.

2. **Важность асинхронности**  
   Использование `async/await` на бэкенде (`FindAsync`, `SaveChangesAsync`) и фронтенде (`fetch`) позволяет не блокировать основные потоки, что критично для отзывчивости приложения.

3. **Локальное состояние упрощает фильтрацию**  
   Хранение всех задач в массиве `allTasks` на фронтенде даёт возможность мгновенно применять фильтры (`active`, `completed`) без повторных запросов к API. Это экономит ресурсы и ускоряет интерфейс.

4. **Защита от XSS — не опция, а необходимость**  
   Функция `escapeHtml()` при вставке пользовательского текста в `innerHTML` предотвращает внедрение вредоносных скриптов. Эту практику нельзя игнорировать даже в учебных проектах.

5. **Миграции автоматизируют развёртывание**  
   Благодаря вызову `db.Database.Migrate()` в `Program.cs` база данных создаётся и обновляется автоматически при запуске API. Это избавляет от ручных операций при переносе приложения на другой компьютер или сервер.

6. **Режимы просмотра / редактирования повышают UX**  
   Переключение видимости блоков `.task-view` и `.task-edit` внутри одной карточки создаёт иллюзию отдельной страницы редактирования, не перегружая интерфейс модальными окнами.

7. **Git + GitHub — неотъемлемая часть разработки**  
   Поэтапные коммиты (модель → контроллер → фронтенд → README) помогают отслеживать прогресс, а публичный репозиторий позволяет показать результат преподавателю или коллегам.
